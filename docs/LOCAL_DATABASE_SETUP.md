# Local Database Setup (Open Server)

## Workflow

GitHub repository is the single source of truth.

Local Open Server copy is used only for:

- running the application;
- testing;
- debugging runtime behavior.

## MySQL Configuration

Current ASU database configuration:

```
driver=mysql
host=127.0.0.1
port=3306
database=asu
username=root
password=
charset=utf8mb4
```

## Create Database

Example:

```sql
CREATE DATABASE asu CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## Apply Migration

Run:

```
database/migrations/001_create_users.sql
```

Example:

```bash
mysql -u root -p asu < database/migrations/001_create_users.sql
```

## Create Administrator

Seed file:

```
database/seeds/001_admin_user.sql
```

Before applying, replace:

```
${ADMIN_PASSWORD_HASH}
```

with a hash generated by ASU PasswordHasher.

## Local Sync

After GitHub changes are complete:

```powershell
git pull origin main
```

Then copy only required runtime files to Open Server.

Do not edit project source directly in:

```
C:\OSPanel\home\asu.local
```

## Verification

Check:

```
/login
/dashboard
/db-health.php
```

Expected flow:

```
POST /login
    -> Authenticator
    -> UserRepository
    -> MySQL users
    -> Session
    -> /dashboard
    -> AuthGuard
```
